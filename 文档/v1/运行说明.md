# 恒德志愿者平台 后端运行说明（前端联调用）

适用制品：`hengde-volunteer-api-1.1-SNAPSHOT.jar`（唯一可启动单元，已聚合 auth / organization / activity / publicity 全部接口）。

---

## 1. 运行前置（本机需安装）

| 组件 | 要求 | 说明 |
|---|---|---|
| JDK | **17** | Spring Boot 4，低版本起不来 |
| MySQL | 8.x | **先手动建一个空库 `hengde_volunteer`**（只建库，不用建表）。默认账号 `root/root` |
| Redis | 任意 5+ | `localhost:6379`，db 0，无密码 |

> 启动时 **Flyway 会自动建全部表（当前迁移到 V17）并写入种子数据**（权限点、超管、组织架构节点），无需手动导 SQL。
> 建空库示例：`CREATE DATABASE hengde_volunteer DEFAULT CHARACTER SET utf8mb4;`

---

## 2. 启动

```bash
java -jar hengde-volunteer-api-1.1-SNAPSHOT.jar --spring.profiles.active=dev
```

> ⚠️ **必须显式指定 profile**：本地联调用 `dev`，生产用 `prod`。
> **不带 profile 会被「生产配置守卫」拦下、直接启动失败**（这是为防止漏配密钥就以开发默认配置上线，属预期行为）。
> 生产用 `--spring.profiles.active=prod` 时，还必须提供真实的 `SECURITY_AES_KEY` / `SECURITY_HMAC_KEY` 和非 `admin123` 的超管密码，否则同样拒绝启动。

连接信息可用环境变量覆盖（不必改 jar）：

| 变量 | 默认 | 用途 |
|---|---|---|
| `DB_USER` / `DB_PWD` | root / root | MySQL 账号密码 |
| `WX_APPID` / `WX_SECRET` | 空 | 微信小程序 appid/secret（**真机微信登录**必需；本地可用开发登录免配，见第 6 点） |
| `SMS_ENABLED` | false | 关 = 验证码只打日志，不真发 |
| `OSS_ENABLED` | false | 关 = 上传返回占位 URL，不真传 |
| `AUTH_SUPER_ADMIN_USERNAME` / `AUTH_SUPER_ADMIN_PASSWORD` | admin / admin123 | 初始超管账号 |

示例（自定义 MySQL 密码）：

```bash
# Windows PowerShell
$env:DB_PWD="你的密码"; java -jar hengde-volunteer-api-1.1-SNAPSHOT.jar --spring.profiles.active=dev
```

> MySQL 主机/库名要改的话，覆盖整条 `spring.datasource.url`：
> `--spring.datasource.url=jdbc:mysql://主机:3306/hengde_volunteer?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai`

## 2.1 用 Docker 跑 MySQL + Redis（推荐，免本机安装）

不想在本机装 MySQL / Redis 的话，用随附的 `docker-compose.yml` 一条命令拉起这两个依赖；**api 仍用 `java -jar` 本机跑**。

前置：装好 **Docker Desktop**（含 docker compose）。

```bash
# 1) 在 docker-compose.yml 所在目录，后台拉起 mysql + redis
docker compose up -d

# 2) 本机启动 api（默认连 localhost:3306 / localhost:6379，正好对上 Docker 映射的端口）
java -jar hengde-volunteer-api-1.1-SNAPSHOT.jar --spring.profiles.active=dev
```

要点：
- compose 里 `MYSQL_DATABASE: hengde_volunteer` 会**自动建空库**，省去手动 `CREATE DATABASE`；表由 Flyway 启动时自动建。
- 账号密码就是默认的 `root/root`，端口映射到本机 `3306`/`6379`，所以 jar 用默认配置即可，**不用设任何环境变量**。
- MySQL 数据挂在命名卷 `hengde-mysql-data`，重启容器不丢。

常用命令：

```bash
docker compose ps          # 看状态
docker compose logs -f mysql   # 看 MySQL 日志
docker compose down        # 停止（保留数据）
docker compose down -v     # 停止并删除数据卷（想清空重来时用）
```

> 注：这里只把 **MySQL/Redis** 放进 Docker；api 不入容器。若哪天想连 api 也容器化，再单独加 api 服务并把 `localhost` 改成容器服务名即可（当前不需要）。

## 3. 访问基址（重要，别漏 `/api`）

## 5. 鉴权方式

- Token 放在 **`Authorization` 请求头**：uuid 风格、**无前缀**、有效期 30 天。
- **志愿者端(`/v`) 与 管理端(`/a`) 是两套独立登录态**，token 不能混用。
- **免登录（公开）接口**：
  - `/v/auth/login/wechat`、`/v/auth/login/dev`（开发登录，dev 限定）、`/v/auth/agreement`、`/v/auth/sms/codes`、`/v/auth/wechat/group-membership`
  - `/a/auth/login`、`/a/auth/sms/codes`、`/a/auth/password/reset`
  - Knife4j 自身（`/doc.html`、`/v3/api-docs/**`、`/webjars/**`）不在 `/v`·`/a` 前缀下，免 token 可开。
- 其余接口都需先登录拿到 token，再放进 `Authorization` 头。

## 6. 登录 / 测试账号（关键）

### 管理端 —— 开箱即用
- 启动时自动创建超管：**`admin / admin123`**
- 调 `POST /a/auth/login` 登录，拿到的 token 用于所有 `/a/**` 接口。

### 志愿者端 —— 本地用「开发登录」免微信（推荐）
- **没有 appid/secret 也能登**：调 `POST /v/auth/login/dev` 直接拿志愿者 token，跳过微信 `code` 换 openid。
  ```json
  { "key": "tester", "registered": true }
  ```
  - `key`：测试身份标识，不同 `key` 对应不同测试志愿者（openid 落为 `dev:{key}`），留空默认 `tester`。
  - `registered: true`：直接造一个**已实名志愿者**，省去走注册流程，便于联调实名后的功能；`false`/不传则是**游客态**（用于测注册流程）。
- **仅 dev 可用**：受 `hengde.auth.dev-login-enabled` 开关控制（dev profile 默认开），生产由「生产配置守卫」fail-fast 拒绝，泄不到线上。
- 走完整注册流程（协议+手写签名）：先 `dev` 登录拿游客 token → `GET /v/auth/agreement` 读协议 → `POST /v/auth/sms/codes` 发码（从控制台日志读）→ `POST /v/auth/register`（`signatureUrl` 必填）。详见 `自测清单.md` 第 5 步。

### 志愿者端 —— 真实微信（上线/真机时）
- `POST /v/auth/login/wechat` 会**真实调用微信 jscode2session**，需要：
  1. 配好真实 `WX_APPID` / `WX_SECRET`；
  2. 微信开发者工具里用真实小程序拿到的 `code`。
- 真机/体验版走这条；本地联调用上面的开发登录即可。

### dev 默认行为（方便测试）
- **短信关闭**：注册 / 找回密码的验证码**只打印到服务端控制台日志**，从日志里读取即可，不会真发短信。
- **实名认证、企业微信群校验关闭 = 放行**：注册时身份证二要素、群成员校验任意值都通过。
- **对象存储关闭**：上传接口返回占位 URL，不真传文件。

## 7. 初始数据为空 —— 先用管理端造数据

库刚启动只有：权限种子、超管账号、组织架构节点。**没有活动 / 公告 / 分队**，所以志愿者端列表、全局搜索一开始都是空的。

建议联调顺序：
1. `admin / admin123` 登录管理端；
2. 发布活动 / 新建公告 / 创建分队；
3. 再测志愿者端的列表、报名、搜索等。

## 8. 已实现 / 未实现范围（以 `url文档v1.md` 为准）

**可联调**：
- auth：志愿者微信登录 / **开发登录**（免微信）/ 实名注册（**协议阅读 + 手写签名**）、管理端账号登录 / 找回密码
- organization：子账号与权限（RBAC）、志愿小组（含批量导入/管理员/解散/组长历史）、归属分队、管理团队标记
- activity：
  - 发布 / 修改 / 删除 / 复制、**固定日期周期发布**、**历史活动发布**
  - 报名 / 取消 / 我的报名 / **同小组代报名**、报名审核与导出
  - **签到 / 时长 / 积分闭环**：现场负责人指派、GPS 签到、统一签退算时长、秘书部确认、积分发放
  - 我的活动、确认到家、双向评价、活动总结
  - **组织部改签到·签退·积分 + 部长二次审核**、**活动留言**、**活动补录 + 部长审核**
- publicity：轮播图、公告、文件下载
- 全局搜索：`/v/search`（活动 + 公告 + 小组 + 分队，合并信息流、下滑分页）

**未实现 / 占位，请勿测**：
- `/v/home` 首页聚合、`data`（数据看板 / 投诉建议）、`user`「我的」（基本信息/志愿者证/证书/地址/安全中心）、`enterprise`（爱心企业）、`donate`（积分兑换/捐赠）、`honor`（排行榜/勋章/证书）、`social`（社区）、活动相册、活动临时负责人考试等。
- 全量功能完成度对照见 `文档/功能清单.md`。

## 9. 安全提醒

弱默认值（`admin123`、`dev-only-*` 加密密钥）**仅限本地联调**。正式部署务必用环境变量覆盖：
`AUTH_SUPER_ADMIN_PASSWORD`、`SECURITY_AES_KEY`、`SECURITY_HMAC_KEY`，并开启 `SMS_ENABLED` / `OSS_ENABLED` 配真实密钥。
