## 四、给需求方测试时的关注点

1. **测试前先在后台建一两个志愿小组**：很多功能（代报名/同组可见/审批）都依赖小组关系
2. **`SMS_ENABLED=false` 时短信只打日志**，注册/找回密码验证码看不见——上线前必须切到 `true` 并把火山密钥配好
3. **`AUTH_REALNAME_ENABLED=false` 时身份证不校验**——真实数据测试前需要先接入腾讯云
4. **PII 加密密钥 `SECURITY_AES_KEY`/`SECURITY_HMAC_KEY`**：第一次写入真实身份证那一刻就锁死了，**永远不能换或丢**——上线前定好、妥善备份
5. **代报名要测**：A、B 在同小组 → A 给 B 报名 → B 在「我的报名」里能看到 → 后台报名列表的「代报名人」列显示 A
6. **签到/时长/积分闭环要全链路测**：发布活动(填经纬度+签到半径) → 指派负责人 → 负责人点活动开始 → 志愿者 GPS 签到 → 负责人统一签退 → 秘书部确认时长 → 发放积分；负责人 ×1.4 / 管理团队 ×1.2 倍率、违规减半/不发要逐一核对
7. **改签到签退积分 / 补录**：组织部改 + 部长二次审核才生效；历史活动补录只得时长不得积分
8. **多端 token 互相不影响**：志愿者端退出不影响后台登录，反之亦然
9. **管理团队志愿者发布活动（V18）+ 发布审核（V19）测全链路**：超管给某志愿者打 `manager_flag=1` → 授 `activity:publish` → 该志愿者在小程序「我的→活动负责人」发布活动 → **活动落「待审核发布」不直接上线**；后台 `activity:publish-audit` 在 `pending-reviews` 看到 → 通过则上线 / 驳回记原因（可 `?status=5` 复查驳回）；**未授权 / 取消标记后再发应被拒（403）**——V18 链路已过 live e2e
