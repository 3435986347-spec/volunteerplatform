# 本地前后端联调指南（用真实接口替换 mock）

本原型所有数据来自 `assets/data.js` 的 `HD.*`（纯前端 mock）。联调时把页面里读 `HD.*` 的地方换成 `await API.get('/a/...')` 即可。已为你准备好统一请求封装 `assets/api.js`。

---

## 一、跑通环境（解决 CORS）

前端是「无构建、浏览器内 Babel」结构，没有 Vite 那种 dev-server 代理，所以**跨域要先解决**。三选一：

### 方案 A（推荐）：前端打包进后端，同源无跨域
把 `恒德志愿者后台.html` + `assets/` 丢进 Spring Boot 的 `src/main/resources/static/`，
后端 `http://localhost:8080/api` 提供接口，页面 `http://localhost:8080/恒德志愿者后台.html`。
此时 `assets/api.js` 里设 `API_BASE = '/api'`（同源，零 CORS）。

### 方案 B：前后端分离 + 后端开 CORS
- 前端：`python3 -m http.server 5500` → `http://localhost:5500/恒德志愿者后台.html`
- 后端：`http://localhost:8080/api`
- `api.js` 设 `API_BASE = 'http://localhost:8080/api'`
- 后端加 CORS（允许 `http://localhost:5500`，允许 `Authorization` 头）：

```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {
  @Override public void addCorsMappings(CorsRegistry r) {
    r.addMapping("/**")
     .allowedOrigins("http://localhost:5500")
     .allowedMethods("GET","POST","PUT","PATCH","DELETE","OPTIONS")
     .allowedHeaders("*")
     .allowCredentials(true);
  }
}
```

### 方案 C：nginx 反代，前端用相对路径
`API_BASE = '/api'`，nginx 把 `/api` 转发到后端 `:8080`，前端静态文件由 nginx 托管。生产常用。

> 不想改 `api.js` 源码也行：在 HTML 里提前注入
> `<script>window.__API_BASE__ = 'http://localhost:8080/api'</script>`（放在引入 api.js 之前）。

## 二、接入 api.js

在 `恒德志愿者后台.html` 里，`data.js` 之后、其它脚本之前，加一行：

```html
<script src="assets/data.js"></script>
<script src="assets/api.js"></script>   <!-- 新增 -->
```

`api.js` 暴露全局 `API`：`API.get/post/put/patch/del`、`API.login`、`API.me`、`API.upload`，
已统一处理 `/api` 前缀、Sa-Token（`Authorization` 头）、响应体 `{code,msg,data}`、401 失效、错误 toast。

## 三、登录链路（先接这个）

把登录页按钮和路由接上真实接口：

```jsx
// app.jsx：登录
async function doLogin() {
  await API.login(account, password);      // 拿 token 存进 localStorage
  const me = await API.me();               // { adminId, realName, department, superAdmin, permissionCodes }
  setIdentity(me);                         // 用真实权限码替换顶栏 mock 身份
  setLogged(true);
}
```

`me.permissionCodes`（超管为 `["*"]`）直接喂给现有的 `hasPerm()`，菜单与 `<Auth>` 按钮自动按真实权限显隐——**这一步打通，整套权限驱动就接上了**。

## 四、把一个列表页从 mock 换成真实接口（以「志愿者管理」为例）

### 改前（读 mock）
```jsx
var rows = HD.VOLUNTEERS.filter(...);          // 前端筛选
var paged = rows.slice((page-1)*size, page*size);
```

### 改后（读后端，筛选/分页交给后端）
```jsx
function VolunteersPage(props) {
  var [list, setList] = useState([]);
  var [total, setTotal] = useState(0);
  var [loading, setLoading] = useState(false);
  var [err, setErr] = useState(false);
  var [page, setPage] = useState(1);
  var [kw, setKw] = useState(''), [gender, setGender] = useState('all'),
      [squad, setSquad] = useState('all'), [political, setPolitical] = useState('all');

  function load() {
    setLoading(true); setErr(false);
    API.get('/a/user/volunteers', {
      keyword: kw,
      gender: gender === 'all' ? '' : gender,
      squad: squad === 'all' ? '' : squad,
      political: political === 'all' ? '' : political,
      page: page, size: 10,
    }).then(function (res) {
      // 后端分页约定 PageResult { records, total }
      setList(res.records || []); setTotal(res.total || 0);
    }).catch(function () { setErr(true); }).finally(function () { setLoading(false); });
  }
  useEffect(load, [page, kw, gender, squad, political]);

  // 表格直接用后端数据 + loading/error 态（Table 组件已支持）
  return /* ... */ React.createElement(Table, {
    columns: cols, data: list, loading: loading, error: err, onRetry: load,
    pagination: { total: total, page: page, size: 10, onChange: setPage },
  });
}
```

### 行操作（写接口）
```jsx
// 禁用/启用
await API.patch('/a/user/volunteers/' + v.id + '/status', { status: to });
// 修改（全量）
await API.put('/a/user/volunteers/' + v.id, form);
// 删除
await API.del('/a/user/volunteers/' + v.id);
// 重置密码
await API.post('/a/user/volunteers/' + v.id + '/password/reset');
// 导出（文件流，直接开新窗口带 token 或后端返回 url）
window.open(API.base + '/a/user/volunteers/export');
// 成功后 message.success(...) + 重新 load()
```

其它页面同理，把对应的 `HD.XXX` 换成 README 里「模块↔接口对照」中的真实路径即可。

## 六、联调顺序建议

1. **接 `api.js` + 登录 + `/a/auth/me`** → 打通鉴权与权限驱动（最关键）。
2. **概览待办计数** → 各 `GET .../pending` 真接口。
3. **逐模块换列表**：志愿者 → 活动 → 报名 → 审核流 → 组织 → 公示。每换一个，删掉对应 `HD.XXX`。
4. **写操作**：approve/reject/PUT/PATCH/DELETE，成功后 toast + 重新拉列表。
5. 全部接完后，`assets/data.js` 可整体删除（仅保留可能复用的状态映射，那些已在 `ui-core.jsx` 的 `MAPS` 里）。

## 七、排错速查

| 现象 | 原因 / 处理 |
|---|---|
| 控制台 CORS 报错 | 后端没开 CORS，或 `API_BASE` 写错 → 见第一节方案 A/B/C |
| 所有请求 401 | token 没带上 / 登录接口返回字段名与 `API.login` 取值不一致 → 调整 `api.js` 里 `tokenValue\|token` |
| 返回数据拿不到 | 响应体不是 `{code:200,data}` → 改 `api.js` 里成功判定与解包 |
| 列表分页错乱 | 后端不是 `{records,total}` → 改页面里 `res.records/res.total` 取值 |
| 上传 415 | 手动设了 `Content-Type` → FormData 必须让浏览器自动设 boundary（api.js 已处理） |
