## 一、跑通环境（解决 CORS）

前端是「无构建、纯 `React.createElement`」结构（M5 起已去运行期 Babel），没有 Vite 那种 dev-server 代理，所以**跨域要先解决**。三选一：

### 方案 A（推荐）：前端打包进后端，同源无跨域
把 `index.html` + `assets/` 丢进 Spring Boot 的 `src/main/resources/static/`，
后端 `http://localhost:8080/api` 提供接口，页面 `http://localhost:8080/index.html`。
此时 `assets/api.js` 里设 `API_BASE = '/api'`（同源，零 CORS）。

### 方案 B：前后端分离 + 后端开 CORS
- 前端：`python3 -m http.server 5500` → `http://localhost:5500/index.html`
- 后端：`http://localhost:8080/api`
- `api.js` 设 `API_BASE = 'http://localhost:8080/api'`
- 后端加 CORS（允许 `http://localhost:5500`，允许 `Authorization` 头）：

```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {
  @Override public void addCorsMappings(CorsRegistry r) {
    r.addMapping("/**")
     .allowedOrigins("http://localhost:5500")
     .allowedMethods("GET","POST","PUT","PATCH","DELETE","OPTIONS")
     .allowedHeaders("*")
     .allowCredentials(true);
  }
}
```

### 方案 C：nginx 反代，前端用相对路径（**生产采用**）
`API_BASE = '/api'`，nginx 把 `/api` 转发到后端 `:8080`，前端静态文件由 nginx 托管。

> 现状：`index.html` 已内置自适应注入——80/443 端口（nginx 同源）用 `'/api'`，其它端口（本地静态服务器）直连 `http://localhost:8080/api`，无需改 `api.js` 源码。

## 三、登录链路（已接通）

登录页与路由走真实接口（`app.js`）：

```jsx
// app.js：登录
async function doLogin() {
  await API.login(account, password);      // 拿 token 存进 localStorage
  const me = await API.me();               // { adminId, realName, department, superAdmin, permissionCodes }
  setIdentity(me);                         // 用真实权限码替换顶栏 mock 身份
  setLogged(true);
}
```

`me.permissionCodes`（超管为 `["*"]`）直接喂给 `authz.js` 的 `hasPerm()`，菜单与 `Auth` 按钮自动按真实权限显隐——这一步打通后，整套权限驱动即为真实数据。

## 四、列表页接接口的统一模式（以「志愿者管理」为例，新增页面照此写）
```jsx
function VolunteersPage(props) {
  var [list, setList] = useState([]);
  var [total, setTotal] = useState(0);
  var [loading, setLoading] = useState(false);
  var [err, setErr] = useState(false);
  var [page, setPage] = useState(1);
  var [kw, setKw] = useState(''), [gender, setGender] = useState('all'),
      [squad, setSquad] = useState('all'), [political, setPolitical] = useState('all');

  function load() {
    setLoading(true); setErr(false);
    API.get('/a/user/volunteers', {
      keyword: kw,
      gender: gender === 'all' ? '' : gender,
      squad: squad === 'all' ? '' : squad,
      political: political === 'all' ? '' : political,
      page: page, size: 10,
    }).then(function (res) {
      // 后端分页约定 PageResult { records, total }
      setList(res.records || []); setTotal(res.total || 0);
    }).catch(function () { setErr(true); }).finally(function () { setLoading(false); });
  }
  useEffect(load, [page, kw, gender, squad, political]);

  // 表格直接用后端数据 + loading/error 态（Table 组件已支持）
  return /* ... */ React.createElement(Table, {
    columns: cols, data: list, loading: loading, error: err, onRetry: load,
    pagination: { total: total, page: page, size: 10, onChange: setPage },
  });
}
```

## 七、排错速查

| 现象 | 原因 / 处理 |
|---|---|
| 控制台 CORS 报错 | 后端没开 CORS，或 `API_BASE` 写错 → 见第一节方案 A/B/C |
| 所有请求 401 | token 没带上 / 登录接口返回字段名与 `API.login` 取值不一致 → 调整 `api.js` 里 `tokenValue\|token` |
| 返回数据拿不到 | 响应体不是 `{code:200,data}` → 改 `api.js` 里成功判定与解包 |
| 列表分页错乱 | 后端不是 `{records,total}` → 改页面里 `res.records/res.total` 取值 |
| 上传 415 | 手动设了 `Content-Type` → FormData 必须让浏览器自动设 boundary（api.js 已处理） |
